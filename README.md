# Hospital Database
## Описание
Проект реляционной базы данных для информационной системы управления больницей.

## Предметная область

Описание пациентов больницы. Включает в себя: 
- карты пациентов
- больничные палаты
- амбулаторные периоды пребывания пациентов в больнице
- результаты обследований, операций и других лечебных или диагностических действий

### Карты пациентов включают:
ФИО, номер, номер страхового полиса (если есть), указания, платно лечится больной или нет, дата создания, адрес пациента (включая индекс, город, улицу, дом, корпус, квартиру), телефоны, паспортные данные пациента (4 составляющих), информация о наличии у пациента аллергии на лекарственные препараты (в виде одного большого текста).

### Больничные палаты включают:
Этаж, номер, отделение, вмещаемое количество больных.

### Амбулаторные периоды включают:
Пациент, дата поступления, причина поступления, дата выписки, выписной эпикриз, палата размещения.

### Результаты включают:
Дата действия, название действия, причина действия, первичный осмотр, результаты действия, назначения, заключения.

Больничные палаты могут относиться к нескольким отделениям. Принадлежность больного к отделению определяется в амбулаторном периоде.

## Состав проекта
- ER-диаграмма (концептуальная модель)
- FA-диаграмма (логическая модель)
- SQL-скрипт (PostgreSQL, создание таблиц без данных)

## Технологии  
- PostgreSQL  
- ERD Tool (pgAdmin 4)  
- draw.io
