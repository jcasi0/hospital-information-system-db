# Hospital Information System Database

## Описание проекта
Проект реляционной базы данных для информационной системы больницы с реализацией аналитических запросов и бизнес-логики на уровне БД.

## Структура базы данных
- пациенты
- палаты
- периоды пребывания пациентов в больнице
- результаты медицинских процедур

## Основные возможности

### Аналитика данных
Реализованы SQL-запросы для анализа:
- диагностической активности (например, ЭКГ за текущий месяц)
- истории назначений
- загрузки больницы (занятость палат)
- количества обследований пациентов

### Бизнес-логика
Реализована на уровне БД:
- ограничения целостности (CHECK constraints)
- триггеры (например, запрет дорогого лечения для "бесплатных" пациентов)
- пользовательские функции
- пользовательские агрегатные функции
- представления (VIEW) и триггеры для работы с ними

## Состав проекта
- `er-diagram/` - ER-диаграмма (концептуальная модель)
- `fa-diagram/` - FA-диаграмма (логическая модель)
- `sql/create_table.sql` - создание структуры БД
- `sql/analysis_queries.sql` - аналитические запросы
- `sql/database_logic.sql` - бизнес-логика (ограничения, функции, триггеры)

## Используемые технологии
- PostgreSQL
- PL/pgSQL
- pgAdmin 4
